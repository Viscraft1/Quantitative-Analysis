# 一键启动说明

## 日常使用

- 双击根目录的 `launch.cmd`
- 或在终端执行 `launch`
- 等待独立应用窗口自动打开
- 关闭后台服务时，双击 `stop.bat`

补充说明：

- `launch.cmd` 是主启动脚本，支持双击一键启动 UI
- `start.bat` 保留为兼容入口，内部会自动转发到 `launch.cmd`
- 首次运行 `start.bat`、`launch.cmd` 或 `python bootstrap.py` 时，会自动安装全局 `launch` 命令
- 安装完成后，在 `cmd` 和 PowerShell 中都可以直接输入 `launch`
- 如果只在当前项目目录临时启动，也可以执行 `.\launch.cmd`
- 如果已经进入 `backend/.venv`，也可以执行 `python -m app.cli launch`

## 首次运行会自动做的事

- 创建 `backend/.venv`
- 安装 `backend/requirements.txt` 里的 Python 依赖
- 检查数据库里是否已有真实市场数据
- 如果没有数据，则自动执行真实数据初始化：
  - 优先导入通达信本地历史日线，拿不到时回退 AkShare
  - 计算因子
  - 生成榜单
- 构建 `frontend/dist` 前端静态页面
- 在后台启动 FastAPI，并通过独立应用窗口打开本地页面

## 默认数据源

`MARKET_DATA_PROVIDER=auto` 时按下面顺序选：

1. 如果存在 `TDX_ROOT`，优先使用本地通达信 `vipdoc`
2. 否则回退到 `AkShare`
3. `Tushare` 建议仅在你显式指定时使用

默认示例配置已经指向：

```env
TDX_ROOT=E:/Tongdaxin/vipdoc
```

## 现在和之前的区别

- 不再依赖演示数据才能打开页面
- 不再需要手动切换到 `backend` / `frontend` 目录
- 不再需要单独启动 Vite 开发服务器
- 页面与接口统一走 `http://127.0.0.1:8000`
- 实时监控页面默认继续使用 `AkShare` 数据源
- 后续再次启动时会复用已初始化环境

## 可选项

- 双击 `create_shortcut.vbs`，会在桌面创建指向 `launch.cmd` 的“启动”快捷方式和“关闭”快捷方式
- 若启动失败，可查看 `.runtime/backend.log`
